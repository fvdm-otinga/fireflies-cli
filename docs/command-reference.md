## Global flags

Every command accepts these flags:

| Flag | Default | Description |
|---|---|---|
| `--profile` | `default` | Config profile name (multi-account) |
| `--json` | false | Shortcut for `--output json` |
| `--output` | `table` | Output format: `table\|json\|ndjson\|yaml\|tsv\|plaintext` |
| `--jq <expr>` | — | Post-process output via embedded gojq expression |
| `--fields a,b,c` | — | Client-side field projection (keep named fields only) |
| `--limit` | 0 | Page size (0 = API default; max 50 for transcripts) |
| `--skip` | 0 | Pagination offset cursor |
| `--transcript` | — | Transcript depth: `none\|preview\|full` |
| `--since` | — | Lower bound filter (RFC3339 or relative like `7d`) |
| `--until` | — | Upper bound filter (RFC3339) |
| `--dry-run` | false | Print the GraphQL op + variables without executing |
| `--yes` | false | Bypass confirmation prompts (destructive ops) |

## Exit codes

| Code | Meaning |
|---|---|
| 0 | Success |
| 1 | General / API error |
| 2 | Usage error (bad flags, missing args) |
| 3 | Auth error (missing or invalid API key) |
| 4 | Rate limit exhausted |
| 5 | Not found |

## Examples

```sh
# List recent meetings as a table
fireflies meetings list

# Get a specific meeting as JSON
fireflies meetings get <id> --json

# View full transcript as speaker-attributed plaintext
fireflies transcript text <id>

# Filter output with jq
fireflies meetings list --json --jq '.[] | {id, title, date}'

# Keep only specific fields
fireflies meetings list --fields id,title,date

# Watch live meetings
fireflies meetings active

# Upload a local file
fireflies meetings upload ./recording.mp3

# Upload from URL
fireflies meetings upload https://example.com/meeting.mp4

# Stream realtime transcript events
fireflies realtime tail <meeting-id>

# Receive webhooks locally (secret read from FIREFLIES_WEBHOOK_SECRET)
FIREFLIES_WEBHOOK_SECRET="$WEBHOOK_SECRET" fireflies webhooks serve --port 8080
```
